# v0.1.0 Release Blockers — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the four issues blocking the v0.1.0 release: two broken documentation references (README code examples, AGENTS.md phantom functions) and two code defects (missing HTTP timeouts, httpx client leak in providers).

**Architecture:** Four independent tasks, no shared state, can run in any order. Tasks 1 and 2 are documentation-only (README.md, AGENTS.md). Tasks 3 and 4 touch connection method and provider source files with corresponding test updates.

**Tech Stack:** Python 3.11+, httpx, pytest, respx, ruff, mypy

**Source:** `.agents/audits/v0.1.0-release-audit.md` (Documentation Drift #1–2, Bugs #1–3, Performance Issues #1–3)

## Global Constraints

- `from __future__ import annotations` at the top of every file.
- Explicit type annotations on all function signatures and module-level variables.
- No bare `except:`. No `except Exception: pass`. Catch specific exceptions.
- No `print()` calls in library code.
- Run `uv run ruff check . && uv run ruff format . && uv run mypy src/authlm && uv run pytest` before each commit.

**Steps:**

- [ ] **Step 1: Add timeout to device-code polling POST**

### Task 4: Fix httpx.AsyncClient Leak in Provider connection_methods()

**Files:**
- Modify: `src/authlm/providers/openai.py`
- Modify: `src/authlm/providers/anthropic.py`
- Modify: `src/authlm/providers/google.py`
- Modify: `tests/unit/test_providers_openai.py`
- Modify: `tests/unit/test_providers_anthropic.py`
- Modify: `tests/unit/test_providers_google.py`

**Interfaces:**
- Consumes: `httpx.AsyncClient` (constructor parameter), `get_oauth_config()` from `_auth_table`
- Produces: `connection_methods(*, include_warned, http_client=None)` — new optional `http_client` parameter on all three providers. Existing callers (registry.py) pass the default `None` and get the current behavior. New callers can pass a managed client.

**Steps:**

- [ ] **Step 1: Write failing tests — verify old behavior still passes**

Run the existing provider tests first to establish baseline:

```bash
uv run pytest tests/unit/test_providers_openai.py tests/unit/test_providers_anthropic.py tests/unit/test_providers_google.py -v
```

Expected: All pass.

- [ ] **Step 2: Add test that verifies client reuse when passed**

Add to `tests/unit/test_providers_openai.py`:

```python
import httpx
from authlm.providers.openai import OpenAIProvider
from authlm.connection_methods.api_key import _default_secret_prompt


def test_connection_methods_accepts_client():
    """connection_methods() uses the passed http_client instead of creating one."""
    client = httpx.AsyncClient()
    provider = OpenAIProvider(secret_prompt=_default_secret_prompt, http_client=client)

    methods = list(provider.connection_methods(include_warned=True))
    # The OAuth methods should have our client, not a fresh one
    for m in methods:
        if hasattr(m, "_http_client"):
            assert m._http_client is client

    # No leak: the provider didn't create its own client
    assert provider._http_client is client
```

- [ ] **Step 4: Modify `OpenAIProvider.connection_methods()` to accept `http_client` param**
