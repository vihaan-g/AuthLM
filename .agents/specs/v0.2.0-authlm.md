# AuthLM v0.2.0 ‚Äî Extensibility & Ecosystem

**Version:** 0.2.0
**Status:** Planned (outline)
**SemVer:** Pre-release ‚Äî API may break.
**Prerequisite:** v0.1.0 shipped.

---

## Theme

AuthLM becomes a platform, not just a library. Third-party plugin authors can add providers, connection methods, and stores via setuptools entry points. The provider count grows from 4 to 20+. The doc site launches. Homebrew install is available.

---

## Scope

### Plugin system

- **pluggy-based plugin system** ‚Äî `hookspecs.py` and `plugins.py` restored and wired through `providers/registry.py`. `pm.hook.register_providers(...)` is the actual provider discovery path, not a hardcoded list.
- **Three hookspecs** ‚Äî `register_providers`, `register_connection_methods`, `register_stores`. Each takes a `register(callable)` callback.
- **First-party providers as default plugins** ‚Äî the 4 v0.1.0 providers register via `@hookimpl` in their respective modules. Loaded via `DEFAULT_PLUGINS` tuple.
- **Third-party plugins via entry points** ‚Äî `[project.entry-points."authlm"]` group. `pm.load_setuptools_entrypoints("authlm")` at startup.
- **Plugin safety rules** ‚Äî lazy SDK imports inside `register_*` or `connect()`, test isolation via `sys._called_from_test`, broken-plugin isolation (caught + logged, not fatal), alias-collision detection (`AliasCollisionError` raised on duplicate `(provider, alias)`).
- **Plugin authoring guide** ‚Äî published at `docs/plugin-authoring.md`. Includes a copy-paste skeleton for a new provider.
- **`ProviderNotAvailable` error** ‚Äî restored. Raised when a provider's SDK extra is not installed. Message includes the install hint (e.g., `pip install "authlm[groq]"`).

### Provider expansion

- **Long-tail providers** ‚Äî added as first-party or community plugins:
  - httpx-only (no SDK needed): GitHub, xAI, Poe, Cerebras, Perplexity, Hugging Face, Moonshot, MiniMax, GitLab, DigitalOcean.
  - SDK-backed: Mistral, Groq, DeepSeek, Cohere, Together, Fireworks.
  - Cloud providers: AWS Bedrock, Azure OpenAI.
- **`OllamaProvider`** ‚Äî no-auth provider for local inference. Adds `AuthMethod.NONE` enum value to `OAuthGrant` (or a new `AuthMethod` enum). The `ConnectionMethod` for Ollama has `warning=None` and `oauth_grant=None`; `connect()` returns a `Credential` with `method_id="none"` and no secret.
- **`AwsCredential`** ‚Äî new credential type for AWS Bedrock. Fields: `access_key_id`, `secret_access_key`, `session_token`. Added to `CredentialUnion`.
- **`AzureAdCredential`** ‚Äî new credential type for Azure OpenAI. Fields: `tenant_id`, `client_id`, `client_secret`, `access_token`, `refresh_token`, `expires_at`. Added to `CredentialUnion`.

### models.dev integration

- **`models_dev.py` restored** ‚Äî live fetch from `https://models.dev/api.json`, on-disk cache at `~/.local/share/authlm/models-dev-cache.json`, vendored fallback snapshot in the wheel.
- **`authlm refresh-registry` command** ‚Äî explicit refresh of the models.dev cache. Fetches, validates, writes to cache. Soft-fails on network error (existing cache untouched, non-zero exit).
- **Providers source metadata from models.dev** ‚Äî `display_name`, `docs_url`, `logo_url` come from `get_provider_metadata(provider_id)` instead of hardcoded values. Auth-specific config (OAuth endpoints, client IDs, scopes) stays in `_auth_table.py`.
- **`scripts/refresh_models_dev.py` restored** ‚Äî operator script to re-vendor the snapshot JSON. Run manually before releases.

### CLI additions

- **`authlm import llm`** ‚Äî one-way migration from `llm`'s plaintext key store (`~/.config/io.datasette.llm/keys.json`). For each `(provider, key)` pair, stores via `authlm.connect()`. Does not delete the `llm` key file.
- **`authlm export [--format json|env] [--output <file>]`** ‚Äî portable plaintext export of all stored credentials. Users are warned that the export file is plaintext. For migration and disaster recovery.
- **Interactive CLI menu** ‚Äî richer method selection (fuzzy search, arrow keys) via `rich` or `questionary`. Replaces v0.1.0's numbered-list prompt.

### Documentation & distribution

- **Doc site** ‚Äî `docs/` directory with `index.md`, `quickstart.md`, `plugin-authoring.md`, `security-model.md`, `api/`. Published to `authlm.readthedocs.io`. `docs.yml` GitHub Actions workflow.
- **`examples/` directory** ‚Äî runnable examples: `connect_api_key.py`, `connect_oauth.py`, `fastapi_integration.py`, `agent_sdk.py`.
- **Homebrew tap** ‚Äî `brew install authlm` on macOS. Separate tap repository (`homebrew-authlm`).
- **`CONTRIBUTING.md` "How to add a provider" section** ‚Äî expanded with plugin authoring guide for v0.2.0.

### Security

- **`Field(repr=False)` on all secret fields** ‚Äî carried from v0.1.0. New credential types (`AwsCredential`, `AzureAdCredential`) follow the same pattern.
- **`redact_body` applied to all OAuth error sites** ‚Äî carried from v0.1.0. New providers' token endpoints use the same redaction.

## Known limitations (carry from v0.1.0)

- **Single-process assumption** ‚Äî multi-process apps refreshing the same credential concurrently may invalidate each other's refresh tokens. File-locking lands in v0.3.0.
- **No proxy/middleware support** ‚Äî AuthLM does not inject credentials into API calls. Use LiteLLM or a local proxy with AuthLM as the credential source.
- **Linux keychain variance** ‚Äî headless Linux without Secret Service falls back to `EncryptedFileStore`.
