# AuthLM v1.0.0 ‚Äî Stable Release

**Version:** 1.0.0
**Status:** Planned (outline)
**SemVer:** Stable ‚Äî breaking changes to the public API require a major version bump. Hookspec signature changes are major bumps.
**Prerequisite:** v0.3.0 shipped.

---

## Theme

API locked. Stability guarantee. AuthLM is production-ready with a long-term SemVer commitment. The public API is frozen, comprehensive integration tests cover every provider √ó method combination, and documentation is complete. All deprecated v0.x APIs are removed.

---

## Scope

### API freeze

- **Public API locked** ‚Äî the following are guaranteed stable through v2.0.0:
  - `authlm.get_credential()`
  - `authlm.get_valid_credential()`
  - `authlm.refresh()`
  - `authlm.should_refresh()`
  - `authlm.connect()`
  - `authlm.validate()`
  - `authlm.set_store()`
  - `CredentialStore` Protocol
  - `Credential` types (`ApiKeyCredential`, `OAuthCredential`, `AwsCredential`, `AzureAdCredential`)
  - `CredentialUnion` discriminated union
  - `parse_credential()`
  - `compute_fingerprint()`
  - `AuthLMError` hierarchy (all subclasses)
  - `Provider` Protocol, `ConnectionMethod` Protocol, `OAuthGrant` enum
  - CLI command structure (`connect`, `list`, `status`, `disconnect`, `env`, `import`, `export`, `audit`, `refresh-registry`)
- **Hookspec signatures locked** ‚Äî `register_providers`, `register_connection_methods`, `register_stores` signature changes are major-version bumps.
- **No new features in v1.0.0** ‚Äî v1.0.0 is a stabilization release. Features land in v1.1.0+.

### SemVer guarantee

- **Breaking changes require major version bump** ‚Äî from v1.0.0 onward, any change to the public API surface (function signatures, exception hierarchy, Protocol shape, CLI command syntax) is a major version bump.
- **Deprecation policy** ‚Äî deprecated APIs emit `DeprecationWarning` for one minor version before removal in the next major version.
- **LTS commitment** ‚Äî v1.0.x receives bugfixes for 12 months minimum. v1.x receives features and bugfixes.

### Comprehensive testing

- **Integration tests** ‚Äî VCR cassettes for every provider √ó method combination. `tests/integration/` fully populated. `pytest-recording` replays cassettes offline.
- **Security tests** ‚Äî `tests/security/` with threat-model-based attack tests: secret leak paths, timing attacks, redaction verification, plugin sandboxing.
- **Coverage gate** ‚Äî `pytest-cov` with `--cov-fail-under=90`. Coverage uploaded to Codecov.
- **CI matrix** ‚Äî 3 OS √ó 4 Python maintained. Added: Python 3.15 when released.

### Documentation complete

- **Every public function documented** ‚Äî docstring with `Args:`, `Returns:`, `Raises:` blocks. Type hints in signatures.
- **Doc site complete** ‚Äî `authlm.readthedocs.io` covers all features: quickstart, API reference, plugin authoring, security model, migration guides.
- **Examples** ‚Äî `examples/` directory with runnable examples for every common use case.
- **CHANGELOG** ‚Äî every release from v0.1.0 to v1.0.0 has a dated section.

### Deprecated API removal

- **All v0.x deprecated APIs removed** ‚Äî any function, class, or CLI flag marked deprecated in v0.7.0 or earlier is removed. Clean break from pre-release cruft.
- **Migration guide** ‚Äî `docs/migrating-to-v1.md` documents every breaking change from v0.3.0 ‚Üí v1.0.0 with before/after examples.

---

## What v1.0.0 is NOT

- **No hosted vault** ‚Äî AuthLM is a developer library, not a SaaS. A hosted credential vault is a different product if there's ever demand.
- **No team/enterprise RBAC** ‚Äî team features, shared credentials, and role-based access control are out of scope. AuthLM is per-user.
- **No web UI** ‚Äî the CLI and Python API are the interface. A web UI for non-technical users is a different product.
- **No TUI** ‚Äî a Textual-based TUI is not planned unless adoption feedback from v0.2‚Äìv0.3 proves the CLI is insufficient.
